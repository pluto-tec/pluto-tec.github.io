与绿子的“不期而遇”

发表于 2018-11-14

与绿子的“不期而遇”

很神奇我恋爱了，如此迅猛从未想到。不过，情形也是四面楚歌，类似一下落入了一口井里，但是自己却异常的开心，暗暗许愿永远不要走出去最好。我知道也感觉到自己失去了理智和全部的决断。为什么呢，那个我生命中曾经出现过重要的人又一次走到了我面前。我想牢牢的永远的握住她的手，我笃定的咬牙默念着。

这几天，无数次想回到过去。回到她大学毕业的时间点，回到她大一我们相见的时间点，回到高三我们分别的时间点，回到高一我们相识的时间点，回到任何一个可能的点。可是，云淡风轻什么也没有变化，什么也没有时空转移，我被无动于衷的按住在此地此时。我想过为什么会这样。自己没有安全感，逃避现实，扭曲真实情感，一系列的人祸。但没什么好抱怨和怪罪的，唯有自己不可饶恕。所以，我和姑娘说要是她想惩罚我，现在是最好的机会，因为我彻底的沦陷了。她淡淡的微笑，一言不发的心疼的抱着我，那眼神和十几年前的一模一样，我动容了久久无法平静内心。抽泣中，心中升腾起一句话，“拜时空交错，叹命运弄人!”

怪又怪我，回到家没两天和母亲谈了此事和想法。这一下子暴风的中心回到我身上！到底是什么，为什么命运要如此折磨我！妈妈几乎崩溃。你说我不理解是假话，我自私任性也不全是。我想过与秋接下来的种种可能和无法估量的困难，我想说无论如何，我想自己选择的去好好的爱一个人，哪怕这一生就这一次机会了，我不想我剩下的人生岁月在后悔和难过中度过，直到一个人消失！

真正好的时间，10年过去了。我们算是在命运的推手下再次相遇。我离婚半年多，她离婚五年儿子八岁刚上小学。听她说了许多这几年的事，每一件我的心都是褶皱的紧绷的渗出紫红色的鲜血。

上周约饭见面，算是同路人的互诉衷肠，当然在我眼里她瘦弱却满怀风骨显得金光闪耀似的。另外，有一点很好奇，也勾起了我全部凌乱的回忆，为什么这一刻的眼神是如此似曾相识，满怀爱恋丝毫不加掩饰。饭后相约，下周一同高原旅行。对我人生从未有过如此说走就走的旅途，事后得知她也一样。我们去了塔尔寺，看了青海湖，游历了张掖丹霞。像所有恋人一样依偎着拥抱着，直到分离时刻的到来，我才明白失去她的意义！




记忆拉回10年前，我还是那个看似高傲冷峻的小屁孩，其实伤了她许多，现在回头看真是造化使然。此时此刻，我在西宁宾馆里，正如2001初冬的夜晚，第一次坐在她床边一样，默默地看着她，守护着她。祈求时间能为我们停下来或者慢下来。可惜后面我们都没有预料到会发生那么多事件，直到我俩在人生的长河里飘飘荡荡了整整17年之久。

写这么多，一是为了记录我们曾经的不朽。二是为了告诉那些年轻的脆弱的恋人们珍惜每一个爱你的灵魂，哪怕就是在心底不愿相信和承认的眼神。人是个敏锐的动物，不要在意任何旁人的指指点点好好去爱一场。世界的包容穿越你的想象，但是时间利刃的伤口难以愈合。不要到了我这个年龄或局面才去懊悔和努力，毕竟你们失去了本来属于你们彼此的美好的青春的时光！

流水账救赎

发表于 2018-11-01

作为一个“混蛋”三周了没有动“笔”，为什么？
事实是自己回顾发现可能是三个月了浑浑噩噩，这里面有我不能言表的苦难和原因.
上个月4号因为一些原因我开始了戒烟。起初的戒断反应在生理上很明显。前3天，脑子里的速度很难形容，类似《头文字D》的一句话“我的眼里全世界都是缓慢的，但是自己却是光速般的移动着”。后面，身体上好挺多，心理的瘾开始上升起来。怎么形容，就是会有另一个自己说，尝试一根也不错，没关系的试试吧。好在有了那两句话。“21天后生理和心理对尼古丁的瘾都会基本消失！（事实的确如此）”，“全世界六大烟草公司一年的纯利润有60亿美金，你还傻里叭唧给人家送钱，帮人家致富！（事实是可以支配的钱的确多了）”
多说一句，当时我的存烟量大概有1w，感谢妈妈帮忙处理了。戒烟也让我明白，“韩笑，你这玩意囤积真是病啊！”。同时，很抱歉大家我的体重又一次上升了而且发展迅速。最后，感谢🌲和母亲的支持和帮助。

妈妈，不得不，说点什么。
或许由于从小和妈妈相依为命惯了，自己会表现的特别随意和放肆，总结来看就是很难长大像个孩子。
最近，莫名其妙的洗衣机事件，我们再一次共同经历和领悟了许多。谢谢你，难以言表的感谢！另外，每一次饭后的对话很精彩很庆幸在这个年纪有了这样的机会去沟通！
当然，介入的太深对我和妈妈都有难以名状的不舒服的感觉，同时父亲一如既往的消失。

和🌲陷入了僵局，还是自己的问题，但是胆小鬼的我不敢面对，只能认命的说“将一切交给命运！” –多么荒唐的说法。

最后，一位十几年没有联系的朋友出现了，说了如此刺入我心的话。“想虚伪就虚伪，何必伪装。不过，你再怎么绕弯我都能撸直。沉迷肉欲并没有错但不能迷失，因为它也是成长的一部分，直到你找到属于自己的真实答案！” –谢谢你朋友，说出了我无法言表的心里话。

就这样流水账般的救赎一下自己，但愿你们能看到。

Hexo中添加本地和网络图片

First
1.把主页配置文件_config.yml里的post_asset_folder:设置为true。

2.在hexo目录下执行：npm install https://github.com/CodeFalling/hexo-asset-image --save（需要等待一段时间）。

3.运行hexo n "xxxx"生成md，/source/_posts文件夹内除了xxxx.md文件还会生成一个同名文件夹。

4.在xxxx.md中想引入图片时，先把图片复制到xxxx这个文件夹中，然后只需要在xxxx.md中按照markdown的格式引入图片：![你想输入的替代文字](xxxx/图片名.jpg)
如：


![test](pic-test/111.png)
![test](https://i.loli.net/2018/02/09/5a7d6a402e15e.jpg)
注意： xxxx是这个md文件的名字，也是同名文件夹的名字。只需要有文件夹名字即可，不需要有什么绝对路径。你想引入的图片就只需要放入xxxx这个文件夹内就好了，很像引用相对路径。

5.最后检查一下，hexo g生成页面后，进入public\2018\08\26\index.html文件中查看相关字段，可以发现，html标签内的语句是<img src="2018/08/26/xxxx/图片名.jpg">，而不是<img src="xxxx/图片名.jpg">。这很重要，关乎你的网页是否可以真正加载你想插入的图片。

这个比较方便！

论沟通

发表于 2018-09-14

忍耐了许久，终于还是到了说两句的时候。

人呀真是活的越久越加敏感，这点从自我保护的角度看很妥帖，但是对他者着实令人痛苦。许多许多时候，就是问了那么一句话，或者不经意间的几个字词我们生活的世界崩塌了。我的结论，当时当刻立马沟通才是正确之路，拖拖拉拉换个法子只会让相处的双方越加困惑。另外，在坚守自我时多留存些爱的眼光或许能浇浇
火。

与家人相处多想想，最终挚爱分离的孤独感，或许能妥善解决许多矛盾。这是我最近学会的道理。

最后说说良师益友。
良师不多见，而且现实工作中几乎不存在，请记住几乎，为什么呢？因为需要经历考验，多数人还没到考验就撒腿便跑了。总结起来，他们有两个特征分享给大家。一则，努力框住你表达自己弱小才能差，只能仰仗你去拼命，其实转了身他和谁都是这么聊的，切莫根据他们的话对号入座，还是兢兢业业对得起职责的完成工作便是了，这里面看清自己和平常心极为重要，算是自己的修炼和考验。二则，给你居高临下感，不会接受新的思想和言论，因为他们身边有所谓可以信任的队友了，你就是后来人。但是，任务分工职责一出天啊全乱了，没思路不成体系该怎么办？适时本着良心和要面对怀疑的把工作做好，切莫多说多谈浪费大家的时间。
“益友”有不多，在你感觉对的时刻往往就是崩塌的边缘。有那么些个时间里，他们要害你你连躲都躲不了，来吧但求还有一口气，回头伤好了也就忘记了那些个卑鄙只为自己利益活着的蚁卵。我以为，真正的朋友是全方面的，生活和工作都有交流交集，诚然我们的家庭各有千秋，但是在一同经历的时刻里，至少是求同存异并肩前行的。感谢你们的存在和支持！

我们的人生，由一个一个的每一天组成，孩提时察觉不到，青年了为家庭为金钱烦恼，中年了为不惑而迷惘，老了也许为健康为下一代操心。很奇妙，我活在夹层里，既不属于你们也不属于他们，有带来的好处，当然也默默的收获着苦痛。
认真真实的过好每一天，不论经历了什么，遇到了好人坏人恶人圣人都如同我窗外的绿树一般，都发自内心出自本心的去沟通去缠绕，因为看似纷杂的当下，我们扮演的角色早已存在早已有了定论。

我爱你们，因为你们曾经出现在我短暂的生命里！

Red Hat Enterprise Linux 7.4上安装Oracle 11.2.0.4

发表于 2018-09-05
  1. 配置Yum源及关闭SeLinux
    [root@localhost ~]# mkdir /media/rhel
    [root@localhost ~]# mount /dev/cdrom /media/rhel
    mount: /dev/sr0 is write-protected, mounting read-only

[root@localhost ~]# vi /etc/yum.repos.d/iso.repo
[iso]
name=iso
baseurl=file:///media/rhel
enabled=1
gpgcheck=0

[root@localhost ~]# vi /etc/selinux/config

modify

SELINUX=disabled

  1. 安装桌面环境（如果未安装）
    [root@localhost ~]# yum -y group install “Server with GUI”

  2. 安装软件包
    [root@localhost ~]# yum -y install binutils binutils-devel compat-libcap1 compat-libstdc++-33 compat-libstdc++-33.i686 ksh elfutils-libelf elfutils-libelf-devel glibc glibc-common glibc-devel glibc-headers gcc gcc-c++ libaio libaio.i686 libaio-devel libaio-devel.i686 libgcc libstdc++ libstdc++.i686 libstdc++-devel libstdc++-devel.i686 make sysstat unixODBC unixODBC-devel libXp libXp-devel libXpm-devel telnet

  3. 建立用户
    [root@localhost ~]# groupadd oinstall
    [root@localhost ~]# groupadd dba
    [root@localhost ~]# useradd -g oinstall -G dba oracle
    [root@localhost ~]# passwd oracle
    [root@localhost ~]# vi /etc/profile

    add to the end

    if [ \$USER = “oracle” ];then

    if [ \$SHELL = "/bin/ksh" ];then
        ulimit -p 16384
        ulimit -n 65536
else
        ulimit -u 16384 -n 65536
fi
umask 022

    fi

  4. 建立文件夹
    [root@localhost ~]# mkdir -p /u01/oracle
    [root@localhost ~]# mkdir -p /u01/oracle/11.2.0/db_1
    [root@localhost ~]# chown -R oracle:oinstall /u01/oracle/
    [root@localhost ~]# chown -R oracle:oinstall /u01/oracle/11.2.0/db_1/
    [root@localhost ~]# chmod -R 755 /u01/oracle/
    [root@localhost ~]# chmod -R 755 /u01/oracle/11.2.0/db_1/

  5. 配置系统核心参数，Oracle用户资源限制，Oracle用户环境变量
    [root@localhost ~]# MEMTOTAL=$(free -b | sed -n ‘2p’ | awk ‘{print $2}’)
    [root@localhost ~]# SHMMAX=$(expr $MEMTOTAL / 2)
    [root@localhost ~]# SHMMNI=4096
    [root@localhost ~]# PAGESIZE=$(getconf PAGE_SIZE)
    [root@localhost ~]# cat >> /etc/sysctl.conf << EOF
    fs.aio-max-nr = 1048576
    fs.file-max = 6815744
    kernel.shmmax = $SHMMAX
    kernel.shmall = $(expr ( $SHMMAX / $PAGESIZE ) * ( $SHMMNI / 16 ))
    kernel.shmmni = $SHMMNI
    kernel.sem = 250 32000 100 128
    net.ipv4.ip_local_port_range = 9000 65500
    net.core.rmem_default = 262144
    net.core.rmem_max = 4194304
    net.core.wmem_default = 262144
    net.core.wmem_max = 1048576
    EOF

[root@localhost ~]# sysctl -p
[root@localhost ~]# vi /etc/security/limits.conf

add to the end

oracle soft nproc 2047
oracle hard nproc 16384
oracle soft nofile 1024
oracle hard nofile 65536
oracle soft stack 10240
oracle hard stack 32768

[root@localhost ~]# vi /home/oracle/.bash_profile

add to the end

export ORACLE_SID=orcl;
export ORACLE_BASE=/u01/oracle;
export ORACLE_HOME=/u01/oracle/11.2.0/db_1;
export PATH=$ORACLE_HOME/bin:$PATH:$HOME/bin
export LD_LIBRARY_PATH=$ORACLE_HOME/lib
[root@localhost ~]# reboot

  1. 编辑主机名
    [root@localhost ~]# vi /etc/hosts

    add

    192.168.10.11 oracle

[root@localhost ~]# vi /etc/hostname

modify

oracle

—[root@localhost ~]# vi /etc/sysconfig/network
NETWORKING=yes
NETWORKING_IPV6=no
HOSTNAME=oracle11g1.localdomain —HOSTNAME为主机名

  1. 解压安装文件
    [root@localhost ~]# cd /opt
    [root@localhost opt]# unzip p13390677_112040_Linux-x86-64_1of7.zip ;unzip p13390677_112040_Linux-x86-64_2of7.zip

  2. 开始安装
    [root@localhost rhel]# xhost +
    access control disabled, clients can connect from any host

[root@localhost rhel]# su - oracle
[oracle@localhost ~]$ export DISPLAY=10.1.105.52:0.0 —此处IP请改为客户端的IP
[oracle@localhost ~]$ cd /opt/database/
[oracle@localhost database]$ ./runInstaller
—检查发现少了一个软件包，且在光盘中搜不到，需要网上下载后手动安装。 [root@localhost opt]# rpm -ivh compat-libstdc++-33-3.2.3-72.el7.x86_64.rpm

[INS-32031]Invalid inventorylocation [INS-32033]Central Inventorylocation is not vritable
解决办法：修改/home/oracle_11的权限chown -R oracle:oinstall /u01

出现错误调用makefile /oracle/app/oracle/product/11.2.0/db_1/sysman/lib/ins_emagent.mk的目标nmo时出错。找到$ORACLE_HOME/sysman/lib/ins_emagent.mk文件
[oracle@localhost ~]$ cd $ORACLE_HOME/sysman/lib
[oracle@localhost lib]$ cp ins_emagent.mk ins_emagent.mk.bak
[oracle@localhost lib]$ vi ins_emagent.mk
查找$(MK_EMAGENT_NMECTL),更改为$(MK_EMAGENT_NMECTL) -lnnz11 —注意：lnnz和$(MK_EMAGENT_NMECTL)之间有空格,然后点“重试“按钮就可以了

在lib包check时，遇到一则告警信息，Package: pdksh-5.2.4 Warning忽略ignore掉就可以。pdksh是一个老包，新的oracle都使用ksh包，但oracle的check机制里面并没有把这个check去掉，所以在执行界面安装oracle的时候，还会有告警信息，我们可以忽略掉它。

[root@localhost opt]# /u01/oracle/oraInventory/orainstRoot.sh
[root@localhost opt]# /u01/oracle/11.2.0/db_1/root.sh
至此安装完成！
